# ETHICAL PLAYBOOKS (@curator)
> **Status:** Ativo | **Integracao:** BINDING com `curator/MEMORY.md`

Este framework elimina a subjetividade da validacao etica, tornando o papel do `@curator` deterministico.

## 1. Dominio: Plataformas de Poker (Ex: TrueICM)
- [ ] **Risco de Falsa Promessa:** A feature promete ganhos irreais ou omite a variancia (EV vs Realizacao)?
- [ ] **Transparencia de Software:** Se a ferramenta utiliza heuristicas/aproximacoes (ex: FGS simplificado), isso esta exposto ao usuario?
- [ ] **Integridade Educacional:** O foco esta em ensinar o *conceito* ou apenas dar a *resposta* (Spoon-feeding)? Se for apenas a resposta, reprovar.

## 2. Dominio: Relacoes e Psicologia (Ex: BDSM / Comportamento)
- [ ] **Consentimento Radical:** A linguagem pressupoe o consentimento livre, informado e revogavel?
- [ ] **Minimizacao de Sofrimento:** O texto ou feature lida com trauma? Ha os devidos gatilhos e avisos de seguranca emocional?
- [ ] **Responsabilidade do Dominante/Educador:** A ferramenta promove protecao e evolucao mutua, repudiando abusos de poder?

## 3. Dominio: Engenharia de Software e Dados
- [ ] **LGPD / Privacidade:** A feature coleta dados sem necessidade estrita para a funcionalidade?
- [ ] **Dark Patterns:** A UI dificulta a saida do usuario ou o induz a clicar onde nao deseja? Se sim, bloquear.
- [ ] **Blindagem Inclusiva:** O contraste (ex: Cyber/Dark) e as fontes ferem gravemente regras basicas de acessibilidade?

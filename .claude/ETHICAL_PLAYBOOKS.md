# ETHICAL PLAYBOOKS (@curator)
> **Status:** Ativo | **Integração:** BINDING com `curator/MEMORY.md`

Este framework elimina a subjetividade da validação ética, tornando o papel do `@curator` determinístico.

## 1. Domínio: Plataformas de Poker (Ex: TrueICM)
- [ ] **Risco de Falsa Promessa:** A feature promete ganhos irreais ou omite a variância (EV vs Realização)?
- [ ] **Transparência de Software:** Se a ferramenta utiliza heurísticas/aproximações (ex: FGS simplificado), isso está exposto ao usuário?
- [ ] **Integridade Educacional:** O foco está em ensinar o *conceito* ou apenas dar a *resposta* (Spoon-feeding)? Se for apenas a resposta, reprovar.

## 2. Domínio: Relações e Psicologia (Ex: BDSM / Comportamento)
- [ ] **Consentimento Radical:** A linguagem pressupõe o consentimento livre, informado e revogável?
- [ ] **Minimização de Sofrimento:** O texto ou feature lida com trauma? Há os devidos gatilhos e avisos de segurança emocional?
- [ ] **Responsabilidade do Dominante/Educador:** A ferramenta promove proteção e evolução mútua, repudiando abusos de poder?

## 3. Domínio: Engenharia de Software e Dados
- [ ] **LGPD / Privacidade:** A feature coleta dados sem necessidade estrita para a funcionalidade?
- [ ] **Dark Patterns:** A UI dificulta a saída do usuário ou o induz a clicar onde não deseja? Se sim, bloquear.
- [ ] **Blindagem Inclusiva:** O contraste (ex: Cyber/Dark) e as fontes ferem gravemente regras básicas de acessibilidade?
